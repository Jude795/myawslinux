PROJECT SUMMARY

This project demonstrates a complete DevOps workflow by automating the build, deployment, and monitoring of a containerized web application using industry-standard DevOps tools on AWS.

The goal of this project is to implement Continuous Integration (CI) and Continuous Deployment (CD) while also setting up real-time infrastructure and application monitoring with automated alerting.

OBJECTIVES
-Automate application build and deployment using Jenkins.
-Containerize the application with Docker.
-Store source code in GitHub.
-Deploy the application on an AWS EC2 instance.
-Monitor server performance using Prometheus and Node Exporter.
-Visualize system metrics through Grafana dashboards.
-Configure Alertmanager to send alerts when server thresholds are exceeded.

TECH USED

-Git & GitHub
-Jenkins

PROJECT WORKFLOW
-Developer pushes code to GitHub.
-Jenkins detects the changes.
-Jenkins pulls the latest source code.
-Docker builds a new image.
-Docker runs the updated container automatically.
-Prometheus collects system metrics from Node Exporter.
-Grafana displays real-time dashboards.
-Alertmanager sends notifications when configured thresholds are reached.
